# Semana de nivelamento
___

## Atividade 01:

1. Crie um arquivo markdown com o nome ```Apresentacao_SEU_NOME.md```
2. Edite o arquivo criado preenchendo o seguinte:

    ```
        Nome:
        Cidade:
        Cursos/Interesses:
        Hobby:
        Expectativa para o curso ADS:
    ```
3. Após finalizar a edição, realize as etapas do fluxo básico do Git (instruções no Notion):
    a. Stage Changes
    b. Commit
    c. Publish Branch
___

## Atividade 02:

Etapa em grupo, simulando a forma como equipes profissionais organizam tarefas em um projeto compartilhado.

>Obs.: Nesta etapa, nenhum arquivo existente deve ser alterado, movido ou excluído. Todo o trabalho será feito apenas criando um novo arquivo em uma branch separada.

1. Cada aluno deve ler a apresentação dos demais integrantes do grupo.
2. O grupo deve entrar em consenso para a escolha de um nome para o time. Em seguida crie uma branch com o nome: ```feature-resumo-NOME-DO-GRUPO```.
3. Criar o arquivo do grupo (somente na branch)
    Crie um arquivo markdown chamado: ```resumo_NOME_DO_GRUPO.md```
    
    Esse arquivo deve conter:
    ```
        Nome do grupo
        Nome dos integrantes
        Algo em comum entre os integrantes (cidade, hobby ou expectativa no curso)
        Uma frase que represente o grupo
    ```

4. Salvar as alterações corretamente
    * Faça o Stage do novo arquivo
    * Realize um Commit com uma mensagem clara
    * Execute o Push da branch criada

5. Unir o trabalho ao projeto principal (Merge)
    * Volte para a branch main
    * Realize o merge da branch ```feature-resumo-grupo```
    * Caso apareça algum aviso ou conflito, chame o professor
